## 💡 提示

### 部署技巧
1. **先测试本地构建**
   ```powershell
   npm run build
   ```
   确保本地构建成功再部署

2. **分步配置环境变量**
   先配置必需的变量，部署成功后再添加可选变量

3. **使用 Vercel 预览**
   每次推送都会创建预览部署，可以先测试预览

4. **查看日志**
   部署失败时，查看 Vercel 日志找出问题

### 常见错误
- ❌ 忘记配置 DATABASE_URL
- ❌ NEXTAUTH_SECRET 未生成
- ❌ Build Command 不正确
- ❌ 环境变量拼写错误
